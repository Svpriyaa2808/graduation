## Design Guidelines

### Color Scheme
- **Primary:** #3B82F6 (Blue)
- **Secondary:** #8B5CF6 (Purple)
- **Success:** #10B981 (Green)
- **Warning:** #F59E0B (Yellow)
- **Danger:** #EF4444 (Red)
- **Background Light:** #FFFFFF
- **Background Dark:** #1F2937
- **Text:** #111827 (Dark) / #F9FAFB (Light)

### Typography
- **Headings:** Inter or SF Pro Display (Bold)
- **Body:** Inter or SF Pro Text (Regular)
- **Code:** JetBrains Mono or Fira Code

### Image Guidelines
- Use high-resolution screenshots (1920x1080 minimum)
- Consistent browser frame (or frameless)
- Show real data in screenshots
- Highlight interactive elements with subtle glow
- Use consistent cursor style in demo screenshots

### Animation Suggestions
- Slide transitions: Fade or Push
- Build animations: Appear or Fade (not too flashy)
- Keep animations under 0.3 seconds
- Use animations sparingly for emphasis

## Presentation Day Tips

1. **Start with impact** - Show the problem first
2. **Demo early** - Get to the demo by slide 10
3. **Tell a story** - Connect features to user benefits
4. **Be enthusiastic** - Show passion for your work
5. **Prepare for questions** - Know your technical details
6. **Have a backup** - Screenshots if live demo fails
7. **Time yourself** - Practice staying within time limit
8. **Engage audience** - Make eye contact, vary tone
9. **End strong** - Emphasize impact and learning
10. **Breathe** - Pause between slides, don't rush
